# Git Worktree Commands

## Create new worktree with new branch
```bash
git worktree add -b <branch-name> ../<directory-name> <base-branch>
# Example:
git worktree add -b feature/video-support ../linux-wall-video main
```

## Create worktree from existing branch
```bash
git worktree add ../<directory-name> <existing-branch>
# Example:
git worktree add ../linux-wall-dev develop
```

## List all worktrees
```bash
git worktree list
```

## Remove a worktreeP
```bash
# First, delete the directory
rm -rf ../linux-wall-video

# Then prune the worktree reference
git worktree prune
```

## Switch between worktrees
Just `cd` into the directory - each is independent:
```bash
cd ../linux-wall-video    # work on video feature
cd ../linux-wall          # back to main
```

## Benefits
- No branch switching (no lost uncommitted changes)
- Run different versions simultaneously
- Test features side-by-side
- Independent node_modules per worktree

## Tips
- Each worktree needs its own `bun install`
- Electron apps can run simultaneously on different ports
- Commit in one worktree, pull in another
- Share the same .git database (saves space)
